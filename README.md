# Phone-Address Microservice

## Запуск с Docker Compose

1. Собрать и поднять контейнеры:

```bash
docker-compose up --build
```

2. Проверить работоспособность:

Документация OpenAPI: http://localhost:8000/docs

Все запросы строго по логике ТЗ.

# ОТВЕТ НА ВОПРОС №2:

Бот сделан довольно архаично - нам предлагается делать отдельный гугл док, куда нужно вставлять
ссылку на репозиторий, и писать ответ на второй вопрос - или писать что ответ в том же репозитории.
Я бы сделал в боте кнопку "Отправить ответ", и дальше отдельные кнопки "Задание №1" и "Задание №2".
При сдаче первого задания можно было бы просто отправлять в бота ссылку на репозиторий, а при сдаче
второго либо сразу текст, либо ссылку на гугл док. Из этого всего бы уже формировался, как вариант,
итоговый док с обеими пунктами, и отправлялся рекрутеру. С точки зрения UX для кандидата это куда
более дружелюбно и приятно.
